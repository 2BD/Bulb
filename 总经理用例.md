## 用例1 审批单据

| **ID** | 1 | **名称** | 审批单据 |
| --- | --- | --- | --- |
| **创建者** | 李培林| **最后一次更新者** | 李培林 |
| **创建日期** | 2017/9/11 | **最后更新日期** | 2017/9/11 |
| **参与者** | 总经理，目标是快速、正确地完成所有单据的审批工作 |
| **触发条件** | 有待审批的单据，即存在处于提交状态的单据 |
| **前置条件** |总经理必须已经被识别和授权 |
| **后置条件** | 更新系统操作日志 |
| **优先级** | 高 |
| **正常流程** | 	1、	系统显示所有待审批单据，按优先级降序排列<br>2、	总经理选择一条单据<br>3、	系统显示单据详细信息<br>4、	查看单据数据是否需要修改，是则进行相应修改<br>5、	对单据进行审批，决定单据是否通过<br>6、	分发到相应部门<br>7、	重复1-6步直到完成所有提交单据的审批 |
| **扩展流程** | 1a、批量审批操作<br>&emsp;1、	批量通过提交状态的单据<br>&emsp;2、	批量驳回提交状态的单据<br>&emsp;3、	批量撤销单据的审批操<br>1b、不存在等待审批的单据<br> &emsp;   1、系统提示无待审批单据<br>4a、撤销单条单据的数据修改<br>   &emsp; 1、将单据数据还原为审批前状态<br>5a、撤销单条单据的审批<br>&emsp;1、	将单据状态从审批后状态退回提交状态 |

## 用例2 制定促销策略

| **ID** | 2| **名称** | 制定促销策略|
| --- | --- | --- | --- |
| **创建者** | 李培林| **最后一次更新者** | 李培林 |
| **创建日期** | 2017/9/11 | **最后更新日期** | 2017/9/11 |
| **参与者** | 总经理，目标是针对不同客户制定促销策略以有效提高业务量 |
| **触发条件** |商品厂家提出自己商品的促销要求；节假日或者竞争促销；提前处理有可能会报废的商品；去除现有的销售策略 |
| **前置条件** |总经理必须已经被识别和授权 |
| **后置条件** | 更新系统操作日志 |
| **优先级** | 高 |
| **正常流程** | 	1、	选择促销策略类别（包括针对客户级别、特价包及针对总价）<br>2a、针对用户级别的促销策略<br>&emsp;1、	选择一个客户级别（Level 1–5）<br>&emsp;2、	选择让利方式<br>&emsp;3a、赠品<br>&emsp;&emsp;1、	输入赠品信息<br>&emsp;3b、价格折让<br>&emsp;&emsp;1、	输入折扣<br>&emsp;3c、赠送代金券<br>&emsp;&emsp;1、	根据面额选择赠送的代金券<br>2b、特价包销售策略<br>&emsp;1、	输入用于特价包的商品信息（编号）<br>&emsp;2、	选择商品数量<br>&emsp;3、	重复1-2步，直到完成所有特价包内商品的输入<br>&emsp;4、	设置降阶幅度<br>2c、针对总价的促销策略<br>&emsp;1、	选择总价区间<br>&emsp;2、	选择让利方式<br>&emsp;3a、赠品<br>&emsp;&emsp;1、	输入赠品信息<br>&emsp;3b、赠送代金券<br>&emsp;&emsp;1、	根据面额选择要赠送的代金券<br>3、设置促销策略的起始时间和间隔时间<br>4、系统建立库存赠送单<br>5、提交等待审批 |
| **扩展流程** | 	2b.2a、商品非法编号<br>&emsp;1、	系统提示错误并拒绝输入<br>2b.3b、总经理从已经选择的商品中删除一个<br>2a.3c.1a、总经理需要一个新的代金券面额<br>&emsp;1、	输入新的代金券面额并创建新的代金券<br>&emsp;2、	选择新的代金券<br>2c.3b.1a、总经理需要一个新的代金券面额<br>&emsp;1、	输入新的代金券面额并创建新的代金券<br>&emsp;2、	选择新的代金券<br>4a、删除一个未提交确认的促销策略 |
