## 用例1 商品分类管理

| **ID** | 1 | **名称** | 商品分类管理 |
| --- | --- | --- | --- |
| **创建者** | 刘瑷玮 | **最后一次更新者** | 刘瑷玮 |
| **创建日期** | 2017/9/11 | **最后更新日期** | 2017/9/23 |
| **参与者** | 库存管理人员，主要是为了方便库存管理 |
| **触发条件** | 有新的类别的商品入库；不再有某类商品的库存；修改某类商品的信息 |
| **前置条件** | 库存管理人员必须已经被识别和授权。 |
| **后置条件** | 记录新的商品分类情况。 |
| **优先级** | 高 |
| **正常流程** | **一．增加分类** <br>1.管理人员申请添加商品种类<br>2. 系统提示输入新的商品种类的名称<br>3.输入新的商品种类的名称 <br>4.从商品种类列表中选择要添加的商品种类的父类<br>5.系统自动生成单据给总经理申请增加种类的操作<br>6.审批成功后系统根据该商品的位置自动为该分类生成一个编号，并在系统中保存<br>7更新系统操作日志<br> **二．删除分类** <br>1.系统显示出所有的商品种类<br>2.	选择要删除的商品种类<br>3.点击要删除的分类进行删除<br>4.系统确认是否删除该分类和该分类的所有子类<br>5.确认<br>6.	再次确认<br>7.系统自动生成单据给总经理审批删除商品种类的操作<br>8 .总经理审批成功，删除该商品种类成功<br>9.系统更新操作日志<br> **三．修改分类** <br>1.系统显示所有的分类列表<br>2.	选择想要进行信息修改的分类<br>3.	选择相应的信息进行更改<br>4.输入相关信息确认完毕<br>5 系统自动生成单据交给总经理进行审批商品种类修改的操作<br>6 总经理审批通过，商品信息修改完成<br>7更新系统操作日志
| **扩展流程** | **一.增加分类** <br> 4a.选择的父类下已经有商品，系统提示是否要更改那些商品的分类<br>&emsp;1.选择更改分类，回到正常流程5<br> &emsp;2.选择不更改分类，此次增加商品种类操作失败，更新系统操作日志，流程结束 <br> 6a.审批没有通过，此次增加商品种类操作失败，更新系统操作日志，流程结束<br> **二.删除分类** <br> 4a.如果要删除的商品种类下仍然有库存信息，禁止此次操作 <br> 8a. 如果审批没有通过，那个删除商品种类的操作失败，更新系统操作日志，流程结束<br> **三.修改商品信息** <br> 3a. 选择进行修改商品种类所在位置<br>4.在商品种类列表中重新选择相应的父类，之后跳转到正常流程的第5步 <br> 5a.总经理审批没有通过，修改商品种类操作结束，更新系统操作日志，流程结束 | 
| **特殊需求** | 1.商品种类的编号和父类编号有关，但一旦生成就不能再改变<br>  2.所有涉及到信息改变的重要环节都必须得到审批<br>3 商品种类信息包括商品种类编号，商品种类名称|

## 用例2 商品管理

| **ID** | 2 | **名称** | 商品管理 |
| --- | --- | --- | --- |
| **创建者** | 刘瑷玮 | **最后一次更新者** | 刘瑷玮 |
| **创建日期** | 2017/9/11 | **最后更新日期** | 2017/9/23 |
| **参与者** | 库存管理人员，为了更加方便的管理库存 |
| **触发条件** | 有新的商品入库；不再有某种商品的库存；修改某种商品的信息 |
| **前置条件** | 库存管理人员必须已经被识别和授权。 |
| **后置条件** | 商品种类的相关信息得到修改 |
| **优先级** | 高 |
| **正常流程** | **一.增加商品** <br>1．库存管理人员提出增加商品的请求<br>2．系统提示从列表中选择该商品的分类<br>3．选择相应的分类<br>4．在系统的提示下依次输入商品的的相关信息，包括编号、名称、型号、库存数量、进价、零售价、最近进价、最近零售价<br>5．提交添加信息<br>6．系统自动生成相应单据交给总经理审批<br>7.总经理审批成功，系统自动生成该商品的编号<br>8.更新系统操作日志<br> **二．删除商品** <br> 1.库存管理人员提出删除商品的请求<br>2.系统显示所有商品的列表，提示选择要要删除的商品 <br>3.选择要删除的商品 <br>4.系统确认是否删除<br>5.系统再次进行确认<br>6.系统自动生成相应的单据交给总经理进行审批<br>7.审批通过，商品删除成功<br> 8.系统更新操作日志 <br> **三．修改商品** <br>1.系统显示所有商品信息<br>2.搜索到想要进行信息修改的商品<br>3.点击该商品并选择进行信息修改<br>4.选择要更改的信息种类<br>5.删除原有信息，输入新的信息<br>&#160;&#160;&#160;&#160;&#160;&#160;重复4-5部直到所有的信息都已经修改完毕<br>6.确认信息修改完毕<br>7.提交修改结果<br>8.系统自动生成单据交给总经理进行审批<br>9.审批通过，系统完成相应信息的修改<br>10.更新系统操作日志<br> **四．查询商品信息** <br>1.系统显示出商品信息界面<br>2.点击查询框<br>3.输入商品的相关信息进行查询<br>4系统显示出商品的查询结果<br>5 选择进行打印商品的目前信息<br>6.更新系统操作日志
| **扩展流程** | **一.增加商品** <br> 4a.如果有些信息没有填入，系统会给出默认值，但商品的名称必须填入<br> 7a.单据审批失败，增加商品种类操作结束，更新系统操作日志，流程结束 <br> **二.删除商品** <br> 3a.如果选择的商品的数量不为0，系统禁止删除该商品<br> 7a. 如果单据审批不通过，删除商品的操作失败，更新系统操作日志，流程结束<br> **三.修改商品的信息** <br> 4a.选择改变该商品的种类<br>&emsp;1.系统显示所有商品种类，选择要添加到的父类<br> &emsp;2转到正常流程的第五步<br>9a.单据审批失败，本次修改商品信息的操作结束，更新系统操作日志，流程结束<br> **四.查询商品信息** <br> 4a.系统找不到该商品的信息，提示用户不存在该商品，更新系统操作日志，流程结束|
| **特殊需求** |1、商品的编号和所在的种类有关，一旦生成就不可以再改变<br>2、商品的属性有编号、名称、型号、库存数量、进价、零售价、最近进价、最近零售价 <br>3.所有涉及到重要信息改变的过程都要进行审批<br> 4. 删除某个商品之后编号仍然保留，历史数据仍然保留，但不能对商品进行任何操作，商品也处于一种不可见的状态<br> 5.修改商品信息时总经理收到的单据包括修改之前和修改之后的信息<br>6.查询商品的时候支持筛选，模糊查询，全查询等多种方式|
