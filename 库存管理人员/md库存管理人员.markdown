## 用例1 商品分类管理

| **ID** | 1 | **名称** | 商品分类管理 |
| --- | --- | --- | --- |
| **创建者** | 刘瑷玮 | **最后一次更新者** | 刘瑷玮 |
| **创建日期** | 2017/9/11 | **最后更新日期** | 2017/9/16 |
| **参与者** | 库存管理人员 |
| **触发条件** | 有新的类别的商品入库；不再有某类商品的库存；修改某类商品的信息 |
| **前置条件** | 库存管理人员必须已经被识别和授权。 |
| **后置条件** | 记录新的商品分类情况。 |
| **优先级** | 高 |
| **正常流程** | **一．增加分类** <br>1.管理人员申请添加商品种类<br>2. 系统显示出所有的商品种类<br>3. 选择新的商品种类要添加的位置   <br>&#160;&#160;&#160;&#160;4a．如果该商品所在的位置是有商品的节点，那么强制将已经有的商品添加到新的分类中<br>&#160;&#160;&#160;&#160;4b.  如果是一个无商品的节点，直接添加分类<br>5.系统提示输新的商品分类的相关信息<br>6.系统根据该商品的位置自动为该分类生成一个编号，并在系统中保存<br>7更新系统操作日志<br>**二．删除分类**<br>1.系统显示出所有的商品种类<br>2.	选择要删除的商品种类<br>3.点击要删除的分类进行删除<br>4.系统确认是否删除该分类和该分类的所有子类<br>5.确认<br>6.	再次确认<br>7.系统成功删除，并同时删除该商品种类的编号<br>8 .更新系统操作日志<br>**三．修改分类**<br>1.系统显示所有的分类列表<br>2.	选择想要进行信息修改的分类<br>3.	点击进行修改<br>&#160;&#160;&#160;&#160;&#160;4a. 点击撤回此次修改操作结束<br>&#160;&#160;&#160;&#160;&#160;4b. 系统显示全部分类<br>5  选择要移动该分类到的地方<br>6  确认进行移动<br>7系统改变该分类的编号<br>8更新系统操作日志
| **特殊需求** | 1、一旦分类下有商品，就不能在该分类下添加子分类 |

## 用例2 商品管理

| **ID** | 2 | **名称** | 商品管理 |
| --- | --- | --- | --- |
| **创建者** | 刘瑷玮 | **最后一次更新者** | 刘瑷玮 |
| **创建日期** | 2017/9/11 | **最后更新日期** | 2017/9/16 |
| **参与者** | 库存管理人员 |
| **触发条件** | 有新的商品入库；不再有某种商品的库存；修改某种商品的信息 |
| **前置条件** | 库存管理人员必须已经被识别和授权。 |
| **后置条件** | 记录新的商品属性。 |
| **优先级** | 高 |
| **正常流程** |**一.增加商品**<br>1．	系统显示所有的商品分类<br>2．选择要增加商品的分类<br>3．点击进行增加商品分类<br>4．依次输入商品的的相关信息，包括编号、名称、型号、库存数量、进价、零售价、最近进价、最近零售价<br>5．提交添加信息<br>6．添加成功，系统自动生成商品分类<br>7.更新系统操作日志<br>**二．删除商品**<br>1. 系统显示所有的商品<br>2. 搜索到要找的商品<br>3.点击删除该商品<br>4.系统确认是否删除<br>5.系统再次进行确认<br>6.商品删除成功，商品的编号也从系统中删除<br>7.更新系统操作日志<br>**三．修改商品**<br>1.系统显示所有商品信息<br>2.搜索到想要进行信息修改的商品<br>3.点击该商品并选择进行信息修改<br>4.选择要更改的信息种类<br>5.删除原有信息，输入新的信息<br>&#160;&#160;&#160;&#160;&#160;&#160;重复4-5部直到所有的信息都已经修改完毕<>br6.确认信息修改完毕<br>7.提交修改结果<br>8.更新系统操作日志<br>**四．查询商品信息**<br>1.系统显示出商品信息界面<br>2.	点击查询框<br>3.输入商品的相关信息进行查询<br>&#160;&#160;&#160;&#160;&#160;&#160;4a系统显示出商品的查询结果<br>&#160;&#160;&#160;&#160;&#160;&#160;4b 系统提示找不到该商品，本次商品的查询结束<br>5  选择进行打印商品的目前信息<br>6.更新系统操作日志<br>
| **特殊需求** |1、点击某个叶节点能显示该分类下的所有商品<br>2、商品的属性有编号、名称、型号、库存数量、进价、零售价、最近进价、最近零售价 |

## 用例5 开库存报损单，报溢单和报警单

| **ID** | 5 | **名称** | 开库存报损单，报溢单和报警单 |
| --- | --- | --- | --- |
| **创建者** | 刘瑷玮 | **最后一次更新者** | 刘瑷玮 |
| **创建日期** | 2017/9/11 | **最后更新日期** | 2017/9/16 |
| **参与者** | 库存管理人员，目标是根据库存中的商品数量做出正确的处理措施 |
| **触发条件** | 库存中商品数量和实际不一致或者商品的数量过少 |
| **前置条件** | 销售人员必须已经被识别和授权。 |
| **后置条件** | 开设相关单据|
| **优先级** | 高 |
| **正常流程** |1 系统显示目前的商品数量<br>2 比较实际的商品数量和系统中商品数量<br>&#160;&#160;&#160;&#160;3a.如果实际商品数量多于系统中商品数量，找出多出的商品类型<br>&#160;&#160;&#160;&#160;3b实际商品数量少于系统中商品类型，找出少的商品类型<br>4. 填写库存报溢表或者库存报损单<br>4 提交单据进行审批<br>5更新系统操作日志
| **扩展流程** |2a如果实际上的商品数量低于商品数量报警线，开库存数量报警单<br>3.更新系统的操作日志|
| **特殊需求** ||
